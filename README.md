# PlaywrightTests

## Overview
This project contains Playwright-based UI tests using MSTest and targets .NET 9.

## Running the Playwright MSTest Suite

### Prerequisites
- .NET 9 SDK installed
- Playwright browsers installed (run `playwright install` or let the test runner download them)

### Run Tests
From the repository root, run:

    dotnet test

To run a single test project or filter tests, use `dotnet test` options or your IDE's test runner.

### Viewing Log Output and Screenshots
The tests write diagnostic output to the MSTest `TestContext`. When running `dotnet test`, you can view logs in the console or open the TRX file generated by the runner.

- To see `TestContext.WriteLine` output in the console, run tests with the default logger (the console logger shows written lines when tests fail or when verbose logging is enabled).
- Screenshots captured on failures are added as result files and saved to the system temp folder; the path is written to the test log.

### Troubleshooting
- If you do not see `TestContext` output, ensure the base test class exposes a public `TestContext` property so MSTest can populate it.
- Ensure Playwright browsers are installed and the CI agent has access to a display or runs browsers in headless mode.

### Notes
- Tests target .NET 9. Adjust your SDK installation accordingly.
